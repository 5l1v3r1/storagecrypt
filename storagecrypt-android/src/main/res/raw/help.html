Keystore creation

The first time you launch the app, a secret key is generated and stored in a secure keystore.

You must provide a password, to protect this keystore. Remember it, you will need to type it every time you launch the app.

It may be tedious, so you have the choice to keep the app running in the background.

This way you won't have to enter your password again, unless you use the "Quit" option in the menu, or shut down your device.

This "Keep alive" parameter can be found, with many others, in the "Settings" menu.

Link your cloud accounts

At first, you only have one storage location where you can encrypt files. It is shown as "Unsynchronized files". As you can guess, the files you encrypt there stay on your device.

If you want your encrypted files to be uploaded to Dropbox, for example, you have to link your Dropbox account first.

To do that, use the button ("Link a new cloud account") at the top of the screen or in the menu.

You can choose between "Google Drive", "Dropbox", "Box", "HubiC" or "One Drive".

You also have to choose which key you want to use by default for this account (you can generate as many keys as you like, but you can also only use the default key).

Then you will have to provide your user name and password for your cloud account (not the password of the keystore).

After linking an account, StorageCrypt will search for any already encrypted files, and download them if any.

Then, you will see a new folder next to the "Local files", which represents your linked account.

Encrypt files

To encrypt a file, you first need to go to the location where you want the encrypted file to be placed, for example "Unsynchronized files".

Then, use the button ("Encrypt") at the top of the screen or in the menu.

You can then encrypt a single file or multiple files. When encrypting a single file, the built in android file chooser will be used to select the file to encrypt.

The "Encrypt multiple files" option uses a custom file chooser, as the built in one does not allow multiple selection.

After file selection, you will have to select which key will be used to encrypt the selected file(s). You can have as many different keys as you like, or use only the default one.

If you encrypted the file inside the "Local files" folder, that's it.

If you encrypted it into a folder related to a cloud account, the file will be uploaded to this account.

Files download and upload

Downloading and uploading files may imply a lot of bandwidth consumption, so you can choose if you want to wait for wifi connection to download, or upload files.

This is the default configuration of the app, so when encrypted, the files will be marked for upload later when you connect to a wifi network.

Note that the app needs to be running to perform uploads or downloads, even if it runs in the background.

If you use the "Quit" button, all running transfers will be stopped, and planned again on next launch.

Create folders

In order to manage your encrypted files more easily, you can create folders on your encrypted storage.

To do that, use the button ("Create en empty folder") at the top of the screen or in the menu.

Give the name of the new folder, and select the key which will be used by default to encrypt files in this folder.

Browse documents

You can navigate through folders by tapping. You can also trigger various actions on folders by touching and holding your finger on them.

On the top of the screen, you will see the name of the current folder. Tap on the icon on its left to go back to the parent folder.

You can also hold your finger on this current folder name to show the other folder actions.

Tap on encrypted files to open them (they will be automatically decrypted on demand). Also, holding your finger on files will show more options.

Manage your keys

If you select the "Manage secret" menu, you will see a list of the existing keys.

Hold your finger on one of them to rename or delete it. Renaming a key doesn't break the files you already encrypted with key.

Warning : if you delete a key, though, you will NOT be able to recover the files encrypted with this key.

In the menu, you will see options to export your keys, for example if you want to access the encrypted files on another device.

The keys you choose to export are saved in a password protected keystore.

Transfer this keystore to your other device, and use the "Import keys" option to import the keys you want.

In the menu, you also have options to create a new key, which you can use to encrypt some files.

The "Change keystore password" menu makes it possible to change the app keystore password. You will have to provide the old password, and the new password.

Settings

If you select the "Settings" menu, you can modify the app settings.

Check the "Keep alive" option if you want to keep the app running in the background.

This way, file transfers will continue event if you open another app, and you won't have to enter your password each time you go open it.

In the "Synchronization" section, you can choose if remote documents actions can be done anytime you are connected, or just if you are connected with wifi.

Remote files deletion don't use much bandwidth, whereas upload and download may use a lot of bandwidth, so choose wisely.

The default configuration is that file deletion will take place with any network type, while upload/download will wait until you are connected with a wifi network.

The last options need a little more explanation.

When you open an encrypted file, StorageCrypt can pass the decrypted data directly to other apps, or decrypt it on a temporary location, and pass them the decrypted file.

The first possibility is more secure, but not all apps support it.

The second option has a little overhead, because the file must me entirely decrypted before using it.

It can be seen as less secure too, because the decrypted file will be written on your device while it is used.

The temporary file will be removed when another one is opened, or whenever you quit the app.

So the "Cache opened files" parameter can be set to "None", "Audio and video files", or "All".

The "None" option will make the app cache no decrypted file on the temporary folder.

The "All" option will make the app cache every file before opening it.

The "Audio and video files" option will make the app cache only audio and video files, because these are the files which often need to be cached before using them.

The last parameter simply lets you choose where these temporary files are decrypted.

The "App internal cache" option is the safest, because, unless your device is rooted, no other app can access it directly, but the space there is also limited by the system, so you might have to choose other ones if you open really big files.

Check remote files

At the top of the main screen, at the right of the current folder name, you will see 2 buttons.

Touching the first one will check if there are new files uploaded on all your remote accounts.

If any, they will be created locally, and a download will be planned. The color of this button will reflect the state of this operation.

The second button reflects the current file synchronisation state (deletion, upload and download).

Touching these buttons will either launch the synchronization action if none is taking place (black indicator), or display the progress window of the running action (any other color).

Note that icons reflecting these states will also appear near each file in the list.

On the progress windows, you usually have two buttons, to cancel the action, or pause it, then resume it later.

Use StorageCrypt from other apps

If you try to open, save or share a file from other applications and the StorageCrypt app is running, you will also see the StorageCrypt folders as sources or destinations.

If you save a file in one of these folders, it will be encrypted with the default key of the directory where you put the file.
